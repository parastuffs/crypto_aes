AES mode CTR.

Library: PolarSSL

If the input data is larger than one block (128 bits), the counter is automatically incremented by PolarSSL.

There is no distinction between the encrypt and decrypt mode.

Parameters

key_sizeKey size in bytes. Either 16, 24 or 32.
nc_strNonce counter string, equivalent to the IV in other modes. 128-bits long.

AES_crypt_type  crypt,
const AES_input *  in,
AES_res *  res 
)

AES general wrapper method.

Switch on the mode and calls the corresponding function. Depending on the mode, some elements of the structures are not needed (e.g. ECB mode does not support IVs). For further details on what is needed, please refer to the specific methods.

Parameters
modeAES mode to use.
cryptEncryption or decryption.
[in]inInput structure. Depending on the mode, not all field need to be filled.
[out]resOutput structure. Depending on the mode, not all fields will be filled.

AES mode XTS.

Library: openssl

Note: bear in mind that in XTS mode, we use two keys. They are concatenated in the key parameter, resulting in a parameter keysize twice as big as the keysize of the mode.

Parameters
cryptENCRYPT or DECRYPT
keyKey string. The two keys used by the mode should be concatenated.
key_sizeKey size in bytes. Either 64 (for 128-bits mode) or 128 (256-bits mode).
[in]ivIV string, 128-bits long.
[in]srcSource string.
src_sizeSource string size, in bytes.
[out]outputOutput string. No need to allocate memory.
[out]output_sizeOutput string size, in bytes. Set in the function.
